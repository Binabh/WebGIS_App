Web GIS

A geographic information system (GIS) is a system designed to capture, store, manipulate, analyze, manage, and present spatial or geographic data. GIS applications are tools that allow users to create interactive queries (user-created searches), analyze spatial information, edit data in maps, and present the results of all these operations.

Web GIS is a type of distributed information system, comprising at least a server and a client, where the server is a GIS server and the client is a web browser, desktop application, or mobile application. In its simplest form, web GIS can be defined as any GIS that uses web technology to communicate between a server and a client.

Types of web services

WMS (Web Map Service)

WFS (Web Feature Service)

WCS (Web Coverage Service)

  • Provides a “simple HTTP interface for requesting georeferenced map images” (OGC)
  • The request defines the spatial extent and layers to be returned
  • A “map” is a digital image of geographic information
  • A map is not the data itself
  • Usually rendered in PNG, JPEG, or GIF format, sometimes SVG
  • Both vector and raster data supported

There are two subclasses of WMS

  • GetCapabilities
  • GetMap
    • Queryable WMS is same as Basic but also supports the optional GetFeatureInfo operation.

The WFS standard defines interfaces and operations for data access and manipulation on a set of geographic features, including:

  • Get or Query features based on spatial and non-spatial constraints
  • Create a new feature instance
  • Get a description of the properties of features
  • Delete a feature instance
  • Update a feature instance
  • Lock a feature instance

While a WMS returns an image of a map, WFS returns the actual features. It can be used with vector data and it allows analysis and editing of the data along with ability to filter and query data. Features are transported in GML format and data may come from multiple sources

WCS is a web service Used to deliver raster data over internet. it is different from WCS because coverages may be used for spatial analysis (whereas a WMS only returns a static image)

Basic WCS operations are:

  • Get Capabilities: Returns an XML document describing the service and coverage within
  • Describe Coverage: Returns an XML document, containing a full description of one or more coverages within a WCS. Provides information such as spatial extent, spatial resolution, cell size, spatial reference system
  • Get Coverage: Returns a requested coverage in a known coverage format (e.g. GeoTIFF) Similar to WMS GetMap and WFS GetFeature requests

Web Services using Open Layer Library

OpenLayers is an open-source JavaScript library for displaying map data in web browsers. It provides an API for building rich web-based geographic applications. OpenLayers supports GeoRSS, KML (Keyhole Markup Language), Geography Markup Language (GML), GeoJSON and map data from any source using OGC-standards as Web Map Service (WMS) or Web Feature Service (WFS). However, there is no built-in support for WCS in OpenLayers and we need to export WCS layers as WMS to view them in browsers.
